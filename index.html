CMG Underwriter Workflow

This interactive workflow is your checklist and reference guide for underwriting loan files at CMG. Each phase includes detailed tasks with links to job aids and resources. Use the checkboxes to track your progress — don't need help on a task? Just check the box and move on!

File Received:
Initial UW Review

SLA: 72 hours from UW Received to Conditional Approval
🏠

Review Cover Letter — Submission Notes

▼
Prior to reviewing the loan, open the Cover Letter – Submission Notes Screen. Review notes from the Processor regarding income calculations, work history, assets, credit items, and any challenging aspects of the file. A paper submission sheet may also be uploaded to Stored Documents.
Cover Letter Review
💡
Tips & Tools: If information is missing or questions arise, check the Conversation Log. Use the New Item icon to add your own notes for the Processor or LO. The Conversation Log icon is available on the toolbar from any screen.
🏠

Review Home Screen & Stored Documents

▼
The Home Screen contains critical loan details. Confirm qualifying FICO, CAIVRs, LDP/GSP, First Time Homebuyer indicators, and property/loan program info. Then review all Stored Documents to ensure every document is either Approved or Inactive with correct expiration dates.
Home Screen
Stored Documents
✊
Reminder: Ensure that every document is approved or inactive! Make sure to add expiration dates for the documents reviewed in Stored Documents.
🏠

Review URLA — Personal Info & Employment/Income

▼
Thoroughly review the URLA Borrower Info Screen. Verify personal information, citizenship status, residence history, and employment/income data. Confirm income calculations match documentation and AUS findings.
Personal Information
Employment & Income
💡
Tips & Tools: CMG Financial Income Worksheets can be found in SharePoint under Operations > Roadmaps, Tools and Product Training. Use the Income Calculation Quick Reference and Self-Employment Income job aids for detailed guidance.
🏠

Review URLA — Assets, Liabilities & Credit

▼
Confirm asset documentation supports funds needed for down payment, closing costs, and reserves. Review all liabilities from the credit report and ensure Real Estate Owned is properly linked and accounted for.
Assets
Liabilities & Credit
Real Estate Owned
🏠

Review / Run AUS Findings

▼
Review AUS findings (DU or LP) and re-run if necessary based on underwriting changes. Ensure all data input to AUS is valid and supported by documentation. For Conventional loans, both DU and LP should be in the file.
AUS Review
Re-Run AUS (if needed)
✊
FHA Reminder: When submitting an FHA loan to AUS, you must choose either DU or LPA. This choice is final and cannot be changed unless the loan type changes. Byte and Clear will lock the FHA AUS choice after the first run.
🏠

Review Appraisal Info Screen

▼
Review the Appraisal Info Screen for completion based on loan program. Confirm the appraised value is set to Actual (not Estimated) before running final AUS. Verify appraisal waivers and property data offerings are documented correctly.
💡
Tips & Tools: Appraisal Review Procedure | Appraisal Policy & Procedure | Appraisal Help Resources — all available on SharePoint/ThinkItHub.
🏠

Review LoanSafe Fraud Report

▼
LoanSafe Fraud Manager analyzes fraud risk and is auto-pulled at File Received. Review the report for alerts and clear as needed. Reports are saved in Stored Documents as Fraud Report. Use LoanSafe Connect for interactive viewing.
💡
Clearing Alerts: Use LoanSafe Connect to clear alerts directly. Notes are stored on the report and sent to the investor. CMG uses "Explained" or "Data Corrected" only as clearing reasons. The score will not change, but alerts will show as cleared for the investor.
🏠

Complete File Quality Review

▼
The File Quality Review screen must be completed after reviewing the initial submission from the Processor. Grade the Processor's submission from A–F. Managers use this feedback for coaching.
💡
Grading Scale: A = Perfect submission, no LO contact needed | B = Acceptable, minimal conditions expected | C = Needed LO clarification before UW | F = Poor submission, restructure required
✅

Approved:
Conditional Approval

Complete 1008, ULDD, conditions, and change status
🏠

Write Conditions (Template & Ad-Hoc)

▼
Add conditions using the Add Conditions from Templates button or create Ad-Hoc conditions for custom requirements. Double-click every condition to review/update for accuracy and fill in any blanks.
Template Conditions
Ad-Hoc Conditions
✊
Important: DO NOT USE SYMBOLS AS LEADING ITEMS in conditions (e.g., **2nd Review Required**). Anything uploaded to the system links to this condition whether it applies or not. If you must use symbols, make sure text comes first.
💡
Condition Color Codes: 🔴 Red = Items submitted but unable to clear | 🔵 Blue = Processing Manager notes | 🟡 Yellow = UW notes for Processor | 🟢 Green = Processor notes for UW | 🟠 Orange = Request to move condition to PTF | 🟣 Purple = UW Manager notes
🏠

Complete 1008 Transmittal

▼
The 1008 Transmittal contains key loan information for the investor. Most fields carry over from other screens, but several items require Underwriter completion.
🏠

Complete ULDD Screen

▼
The ULDD screen replaces the LQA run for Conventional loans. It is the Underwriter's responsibility to complete all possible information to ensure the file can be processed post-closing.
🏠

Update Underwriting Screen & Change Status

▼
The Underwriting Screen provides a general summary of the loan. Complete expiration dates, verify AUS info, and enter all required fields before changing the loan status.
Underwriting Screen
Change Status
🏠

Mortgage Insurance & Second Level Review

▼
Review MI requirements for loans with LTV > 80%. The Processor is responsible for ordering MI, but the Underwriter must verify the MI cert matches the loan terms. A Second Level Review is required for loans with higher risk criteria.
Mortgage Insurance
Second Level Review (If Required)
🔍

Conditions Review:
Clear to Close

CTC Metric: Clear to close received 48 hours prior to closing (70% of the time)
🏠

Clear Prior-to-Docs Conditions

▼
When a file is submitted as Conditions for Review, go to the Conditions screen. Documents linked to conditions display a yellow folder icon. All PTD conditions must be cleared for CTC.
🏠

Review UDM & Final Compliance

▼
Before issuing CTC, review the Undisclosed Debt Monitoring report and ensure ComplianceEase passes. Any new debts require re-underwriting.
Undisclosed Debt Monitoring
ComplianceEase & FEMA
🏠

Validate ULDD & Final AUS

▼
Prior to issuing CTC, validate the ULDD screen and cure all errors. Print final AUS findings and all documents to Stored Documents.
🏠

Issue Clear to Close

▼
Once all PTD conditions are cleared, the file is ready for Clear to Close. Change the status and the file advances to the Closing team.
✊
Suspended TBD Files: Suspended TBD files are NEVER to be placed in Approved status. ALWAYS use Pre-Approved.
📚
